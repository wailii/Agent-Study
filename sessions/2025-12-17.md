# 学习笔记 - 2025-12-17

## 📌 Core Topic

B.1 ReAct (Reason+Act) 模式 & 在 RAG / 需求文档助手中的架构设计

## 💡 My Takeaway

- ReAct = Thought → Action → Observation 的循环,让模型边观察环境边决定下一步,而不是一次性脑补完整流程
- history/短期记忆很关键,可以复用之前的尝试结果,避免在多轮检索/调用工具时重复踩坑
- ReAct 适合作为“高开销模式”服务复杂任务,前面应加一层意图识别/路由,简单问题直接用单轮 LLM
- 在工程上需要 Loop Controller 控制步数/时长,结合缓存、防死循环和停止原因记录
- 通过在 ReAct 后增加 Critic/Reflection 层,可以在给用户前检查是否答题、是否有文档证据、避免幻觉
- 针对需求文档助手场景,已经能设计出: 意图识别 → ReAct+需求工具层 → 验证器 的整体数据流
